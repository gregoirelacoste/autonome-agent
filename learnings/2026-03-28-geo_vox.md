## 1. Problèmes identifiés & Raison d'être

### 1.1 Timeouts et blocages longs
- **Symptôme :** `research-initial` phase a duré 18m26s (Feature 1.2) → logs "Claude semble bloqué"
- **Raison :** Phase unique de 80 turns pour exploration complète du projet (overkill initial)
- **Impact :** Augmente les coûts API, risque timeout Vercel, réduit feedback immédiat

### 1.2 Limite de turns atteinte
- **Symptôme :** Feature 1.2 → "Claude a atteint la limite de turns (50) en phase 'implement'"
- **Raison :** max_turns=50 insuffisant pour features complexes (Job Queue 3.2 nécessita 5 turns)
- **Impact :** Implémentation potentiellement incomplète sans signal clair au user

### 1.3 Rétrospectives échouées (exit=1)
- **Symptôme :** Feature 1.1 → "Rétrospective échouée — on continue"
- **Raison :** exit=1 mais pas de logging du motif (timeout? exception?) → blind retry
- **Impact :** Perte de feedback critique ; patterns non documentés ; pas de data pour future improvement

### 1.4 Auto-map.md surdimensionné
- **Symptôme :** `.orc/codebase/auto-map.md` = 202 lignes générées systématiquement
- **Raison :** Export TS exhaustif + contexte complet injecté avant chaque feature
- **Impact :** Bloat de contexte, consommation tokens unnecessaire, confusion au lieu de clarté

### 1.5 Pas de guard-fou pour états orphelins
- **Symptôme :** Lock orphelin détecté (PID 988985 mort) → cleanup manuel
- **Raison :** Aucun housekeeping systématique au démarrage
- **Impact :** Sessions fantômes, état desynchronisé, raceconditions potentielles

## 2. Recommandations par phase

### Phase 1 : Recherche initiale (`research-initial`)

**Problème :** 80 turns pour explorer codebase de 0 → overkill, génère faux positifs
**Solutions :**

1. **Splitter recherche en sous-phases** (max 20 turns each)
   - `research-brief` : Parser CLAUDE.md, BRIEF.md uniquement (2-3 turns)
   - `research-arch` : Lire architecture.md, data-models.md, ADRs (3-4 turns)
   - `research-codebase` : Outil Glob/Grep smart pour fichiers clés (2-3 turns)
   - Éviter : énumération exhaustive de tous les fichiers

2. **Timeout par sous-phase**
   - Chaque sous-phase : max 5min (vs 18min actuellement)
   - Si une sous-phase timeout → skip et log avertissement, continue

3. **Cache recherche**
   - Sauvegarder l'état du `.orc/state.json` après recherche initiale
   - Skip `research-initial` si projet existant ET `state.json` présent (sauf --fresh flag)

**Config proposée (config.sh):**
```bash
RESEARCH_PHASES=("research-brief" "research-arch" "research-codebase")
RESEARCH_TIMEOUT_PER_PHASE=300  # 5 min
RESEARCH_CACHE_ENABLED=true
RESEARCH_SKIP_IF_STATE_EXISTS=true
```

### Phase 3 : Rétrospective (`reflect`)

**Problème :** exit=1 sans raison → blind continue, patterns perdus
**Solutions :**

1. **Logging structuré des raisons de failure**
   ```bash
   # Avant (silencieux) :
   # → Claude lancé [phase=reflect] ... exit=1

   # Après (diagnostic) :
   # → Claude lancé [phase=reflect] ...
   # → Motif: TIMEOUT | EXCEPTION (type) | MAX_TURNS
   # → Sortie tronquée : {retro-fichier} n'a pas terminé
   ```

2. **Fallback graceful si reflect échoue**
   - Générer une rétrospective minimale (skeleton) avec :
     - Status feature (merged OK, tests passed)
     - Metrics brutes (fichiers modifiés, lignes +/-, coût)
     - ⚠️ "Rétrospective incomplete — voir logs pour détails"
   - Ne PAS continuer silencieusement (perte d'apprentissage)

3. **Retry reflect une fois sur timeout**
   ```bash
   REFLECT_TIMEOUT=180  # 3 min (vs inline actuellement)
   REFLECT_RETRY_ON_TIMEOUT=true
   REFLECT_FALLBACK_MINIMAL=true
   ```

## 4. Conventions bash à ajouter

### 4.1 Exit codes distinctifs
```bash

# Failures

ERROR   # Failure (mais continue)
FATAL   # Abort (orchestrator s'arrête)

[2026-03-28 14:02:34] [WARN] [implement] [Feature 1.2] Approche limite: 48/50 turns

```

## 9. Versions Bash Conventionnelles (pour stack-conventions-bash.md)

### Exit Codes Distinctifs (NOUVEAU)

```bash
exit 0    # Success
exit 1    # Generic failure (build failed, test failed)
exit 2    # TIMEOUT (phase exceeded max time)
exit 3    # PARSE_ERROR (can't parse subprocess stdout/stderr)
exit 4    # GIT_ERROR (merge/commit/push failed)
exit 5    # STATE_CORRUPTED (state.json invalid or unreadable)
exit 42   # MAX_TURNS_REACHED (specific to implement phase — signal retry)
```

### Logging Format (Structured)

```bash
# [TIMESTAMP] [LEVEL] [PHASE] [FEATURE] message
# TIMESTAMP = ISO 8601 time
# LEVEL = INFO | WARN | ERROR | FATAL
# PHASE = research-brief, research-market, planning, implement, test, reflect
# FEATURE = feature ID (e.g., 1.1, 3.4)

log_info()  { echo "[$(date +%Y-%m-%d\ %H:%M:%S)] [INFO]  [$1] [$2] $3"; }
log_warn()  { echo "[$(date +%Y-%m-%d\ %H:%M:%S)] [WARN]  [$1] [$2] $3" >&2; }
log_error() { echo "[$(date +%Y-%m-%d\ %H:%M:%S)] [ERROR] [$1] [$2] $3" >&2; }
log_fatal() { echo "[$(date +%Y-%m-%d\ %H:%M:%S)] [FATAL] [$1] [$2] $3" >&2; exit 1; }
```

### Subprocess Pattern (Timeout + Exit Code Handling)

```bash
run_claude_phase() {
  local phase="$1" feature="$2" max_turns="$3" timeout_sec="$4"

  log_info "$phase" "$feature" "Spawning Claude (turns=$max_turns, timeout=${timeout_sec}s)"

  timeout "$timeout_sec" \
    npm run claude -- --phase="$phase" --feature="$feature" --max-turns="$max_turns" \
    > ".orc/logs/${phase}-${feature}.log" 2>&1

  local exit_code=$?

  case $exit_code in
    0)   log_info "$phase" "$feature" "Success"; return 0 ;;
    2)   log_error "$phase" "$feature" "TIMEOUT (>${timeout_sec}s)"; return 2 ;;
    42)  log_warn "$phase" "$feature" "Max turns reached"; return 42 ;;
    *)   log_error "$phase" "$feature" "Failed with exit $exit_code"; return $exit_code ;;
  esac
}
```

### Atomic State Updates (Lock-protected)

```bash
# Use flock + jq for safe JSON updates
update_state_atomic() {
  local key="$1" value="$2"

  # Backup before update
  cp .orc/state.json .orc/state.json.bak

  # Update safely with jq
  jq --arg key "$key" --argjson value "$value" \
    '.[$key] = $value' \
    .orc/state.json > .orc/state.json.tmp

  # Validate JSON before swapping
  if ! jq . .orc/state.json.tmp > /dev/null 2>&1; then
    log_error "state" "update" "Invalid JSON, rolling back"
    rm -f .orc/state.json.tmp
    return 1
  fi

  # Atomic swap
  mv .orc/state.json.tmp .orc/state.json
  log_info "state" "update" "State updated: $key"
  return 0
}
```

### Lock Management (Cleanup Orphans)

```bash
cleanup_orphan_locks() {
  if [[ -f .orc/.lock ]]; then
    local pid=$(cat .orc/.lock)
    if ! kill -0 "$pid" 2>/dev/null; then
      log_warn "startup" "orphan-lock" "PID $pid dead, cleaning"
      rm -f .orc/.lock .orc/.pid .orc/.watch-pid
      return 0
    fi
  fi
  return 0
}

# Call at startup
trap "cleanup_orphan_locks" EXIT
cleanup_orphan_locks
```

### Retry Pattern (Exponential Backoff with Exit Code Handling)

```bash
run_with_retry() {
  local phase="$1" feature="$2" max_retries="${3:-2}"
  local attempt=1

  while [[ $attempt -le $max_retries ]]; do
    log_info "$phase" "$feature" "Attempt $attempt/$max_retries"

    run_claude_phase "$phase" "$feature" 60 300
    local exit_code=$?

    case $exit_code in
      0) return 0 ;;
      42) # MAX_TURNS_REACHED
        if [[ $attempt -lt $max_retries ]]; then
          local extra_turns=$((60 + (20 * attempt)))
          log_warn "$phase" "$feature" "Retrying with $extra_turns turns"
          ((attempt++))
          continue
        fi
        ;;
      *) return $exit_code ;;
    esac

    ((attempt++))
  done

  log_fatal "$phase" "$feature" "Failed all $max_retries attempts"
  return 1
}
```

## 10. Anti-patterns Bash à Éviter

### ❌ INCORRECT

```bash
# ❌ Ignoring errors silently
cmd || true

# ❌ Hardcoded values
TIMEOUT=600

# ❌ Fire-and-forget subprocess
npm run build &

# ❌ Parsing JSON with grep/sed
grep "status" state.json | sed 's/.*: "\(.*\)".*/\1/'

# ❌ Race condition on state.json
echo "$NEW_STATE" >> .orc/state.json

# ❌ No secret redaction in logs
echo "API_KEY=sk_test_123456" >> .orc/logs/test.log
```

### ✅ CORRECT

```bash
# ✅ Explicit error handling + logging
if ! cmd; then
  log_error "phase" "task" "Command failed with exit $?"
  return 1
fi

# ✅ Config-driven values
source config.sh
timeout="$GLOBAL_TIMEOUT"

# ✅ Subprocess tracking with reap
cmd > output.log 2>&1 &
PID=$!
wait $PID
exit_code=$?

# ✅ Safe JSON parsing
status=$(jq -r '.status' .orc/state.json)

# ✅ Atomic state updates
update_state_atomic "status" "completed"

# ✅ Redact before storage
sed -i 's/sk_test_[a-zA-Z0-9]\+/sk_test_****/g' .orc/logs/test.log
```
